# システム仕様書 (SPEC.md)

このドキュメントでは、`bunka` のソフトウェア要件、インターフェース定義、アルゴリズムの詳細、およびGUI設計仕様について説明します。

---

## 1. CLI システム仕様

### 1.1 インターフェース定義
本ツールはコマンドラインインターフェース（CLI）で動作し、引数に変換対象の小数点数とオプションパラメータを指定できます。

- **コマンド構文**:
  - 数値変換時: `bunka <小数点数> [オプション]`
  - バージョン表示時: `bunka --version` (または `-v`, `-V`)
  - ヘルプ表示時: `bunka --help` (または `-h`)
- **標準出力 (stdout)**:
  - 数値変換時: 既約分数の形式 `<分子>/<分母>` （例: `1/7`）で出力されます。
  - バージョン表示時: `bunka <バージョン番号>` （例: `bunka 0.2.0`）で出力されます。
  - ヘルプ表示時: 使用方法、引数、オプション、使用例を含むヘルプメッセージが出力されます。
- **標準エラー出力 (stderr)**: 実行失敗時、または引数が無効な場合にヘルプや警告を出力します。
- **終了コード (Exit Codes)**:
  - `0`: 変換成功、バージョン情報表示成功、またはヘルプ表示成功。
  - `1`: 変換失敗（引数不足、または無効な浮動小数点数値、未知のオプション指定）。

### 1.2 パラメータオプション一覧
CLIの引数の後ろに以下のパラメータを指定して、分数近似アルゴリズムの制御上限を変更することができます。

| オプション（ロング） | オプション（ショート） | 値の型 | デフォルト値 | 役割 |
| :--- | :--- | :--- | :--- | :--- |
| `--max-den` | `-d` | 正の整数 (`u64`) | `100,000` | 近似計算に使用する最大分母。 |
| `--tolerance` | `-t` | 正の浮動小数点数 (`f64`) | `1e-6` | 近似計算の許容誤差しきい値。 |

#### コマンド指定例:
```bash
$ bunka 0.142857 -d 1000 -t 1e-4
1/7
```

### 1.3 アルゴリズム仕様（連分数展開）
実数 $x$ を分数 $h/k$ に近似するために、`bunka` は**連分数展開（Continued Fraction Expansion）**アルゴリズムを採用しています。

1. **符号の処理**: 入力値の符号を判定・抽出します。絶対値 $|x|$ に対して分数を算出し、最終的に元の符号を分子に乗算します。
2. **展開ループ**:
   - 残差 $r_n$（初期値は $|x|$）の整数部分を $a_n$ とします。
   - 以下の漸化式を用いて近似分数の数列（近似分数）を動的に計算します：
     $$h_n = a_n h_{n-1} + h_{n-2}$$
     $$k_n = a_n k_{n-1} + k_{n-2}$$
     ここで初期状態は $(h_{-1}, k_{-1}) = (1, 0)$ および $(h_{-2}, k_{-2}) = (0, 1)$ と定義されます。
3. **終了判定**:
   - 計算された分母 $k_n$ が最大許容分母 $max\_denominator$ を超えた場合、ループを終了し、手前の近似分数 $h_{n-1}/k_{n-1}$ を採用します。
   - 対象値 $x$ と現在の近似値 $h_n/k_n$ の絶対誤差が許容誤差 $tolerance$ 以下になった場合、即座にループを終了します。
   - 残差の小数部分（$r_n - a_n$）がほぼゼロ（`10^-11` 未満）になった場合、ループを終了します。
   - ループ回数が `50` 回に達した場合、異常ループ防止のため強制終了します。
   - 残差を更新します： $r_{n+1} = 1 / (r_n - a_n)$。

## 2. GUI システム仕様

[src/main.rs](../src/main.rs) は Cargo features に `gui` を指定してビルドすることで、最前面常駐のデスクトップGUI版として起動させることができます。

- **ビルドコマンド**:
  ```bash
  cargo run --features gui
  ```

### 2.1 UI/UX 要件
- **オーバーレイパネル構成**:
  - **小数点数入力**: ユーザーが直接テキスト入力可能で、変更時に即座に結果を再計算します。
  - **最大分母スライダー**: 近似計算に使用する分母の上限（`10` 〜 `100,000`）を設定します。
  - **許容誤差スライダー**: $10^{-9}$ から $10^{-1}$ までの範囲で許容誤差を指数指定します。
  - **結果表示エリア**: 変換された分数を大きな文字で中央に強調表示します。

### 2.2 ネイティブウィンドウフレームワークの制約 (eframe/egui/windows)
[.agents/AGENTS.md](../.agents/AGENTS.md) に定義されたガイドラインに基づき、デスクトップ版のウィンドウは以下の制約を実装しています。
- **ウィンドウ枠および影の非表示**:
  - `NativeOptions` で `decorated: false` および `transparent: true` を指定して、OS標準の枠を無効化しています。
  - `windows` クレートを用いて Named Mutex を取得し、二重起動防止を行っています。すでにプロセスが起動している場合は、新しく起動されたプロセスが即座に正常終了します。
- **枠なしウィンドウのドラッグ移動**:
  - タイトルバーを持たないため、最上部の見出しエリアなどをドラッググリップとします。
  - グリップがドラッグされた際、`frame.drag_window()` を呼び出すことでネイティブウィンドウを移動させています。
