# フットプリント記録 (FOOTPRINTS.md)

このドキュメントでは、`bunka` のリリースビルドされた実行ファイルのファイルサイズや実行性能（フットプリント）に関する実測データを追跡・記録します。

---

## 1. 測定指標の概要

以下の測定データは、Windows 11 x64 環境で Rust 1.80（または edition 2024 対応コンパイラ）を使用し、`x86_64-pc-windows-msvc` ターゲット向けにビルドした結果に基づいています。

### 1.1 バイナリサイズ
- **ファイルサイズ**: `139,776 バイト` (約 136.5 KB)
- **評価**: 非常に優れています。厳格なリリース最適化オプションを適用することにより、標準的な Rust 実行ファイルとして極限に近いサイズ削減を達成しました。

### 1.2 メモリ使用量 (RAM)
- **ピーク時の物理メモリ使用量 (Working Set)**: 実行中はおおよそ `1.2 MB 〜 2.0 MB` と推定。
- **ライフタイム**: 近似計算処理の過程において、すべての計算はスタック割り当ての状態遷移のみで完結します。アルゴリズムのメインループ内部ではヒープ割り当て（`Vec`, `String` など）を一切行いません。

### 1.3 実行時間
- **起動および計算完了までの時間**: `約 20 ミリ秒` (PowerShell の `Measure-Command` による測定)。
- **アルゴリズム展開ステップ数**: 最大 50 サイクル（通常の有理数に近い小数点数では 3 〜 10 サイクル程度で収束します）。

---

## 2. リリース最適化設定

上記の極小サイズおよび高速な動作は、[Cargo.toml](../Cargo.toml) の `[profile.release]` セクションに適用された以下の設定によるものです：

```toml
[profile.release]
opt-level = 'z'       # バイナリサイズを最優先で最適化（命令の配置を圧縮）
lto = true            # リンク時最適化（デッドコードの削減をクレート間で徹底）
codegen-units = 1     # コード生成単位を1に統合（LLVMによるインライン化を最大化）
panic = 'abort'       # パニック時に即時終了（スタック展開用のメタデータと展開ロジックを排除）
strip = true          # シンボル情報とデバッグ情報を実行ファイルから完全に削除
```

---

## 3. 測定・再現手順

自身の環境でこれらのフットプリントを測定または再現するには、**PowerShell 7 (pwsh)** を起動し、以下のコマンドを実行してください。

### ファイルサイズの測定
```powershell
(Get-Item target/release/bunka.exe).Length
```

### 実行時間の測定
```powershell
Measure-Command { target/release/bunka.exe 0.142857 }
```
